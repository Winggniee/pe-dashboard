# 📊 工作负载计算系统

## 概述

新的工作负载计算系统结合了**项目难度**和**客户状态**两个维度，提供更精确的PE工作量评估。

## 计算公式

```
项目负载 = 难度权重 × 状态倍数
PE总负载 = 所有项目负载之和
```

## 默认权重配置

### 📈 项目难度权重

| 难度等级 | 默认权重 | 说明 |
|---------|---------|------|
| 高难项目 | 3.0 | 技术复杂、耗时长的项目 |
| 中等项目 | 2.0 | 标准难度项目 |
| 简单项目 | 1.0 | 简单快速的项目 |

### 🔄 客户状态倍数

| 状态 | 默认倍数 | 工作负载程度 | 说明 |
|-----|---------|-------------|------|
| 调优中 | 1.5 | 🔴 高 | 需要持续优化调整，工作量大 |
| 搭建中 | 1.5 | 🔴 高 | 初期搭建阶段，需要大量投入 |
| 维护中 | 1.0 | 🟡 中等 | 日常维护，标准工作量 |
| 测试中 | 1.0 | 🟡 中等 | 测试验证阶段 |
| 未知状态 | 0.5 | 🟢 低 | 状态不明确 |
| 等待中（等商务） | 0.3 | 🟢 低 | 等待商务响应，PE工作量小 |
| 等待中（等客户） | 0.3 | 🟢 低 | 等待客户响应，PE工作量小 |
| 封号 | 0.1 | ⚪ 极低 | 账号被封，几乎无工作量 |
| 已流失 | 0.1 | ⚪ 极低 | 客户流失，几乎无工作量 |

## 使用示例

### 示例1: 高难度维护项目
```
难度: 高难项目 (3.0)
状态: 维护中 (1.0)
负载 = 3.0 × 1.0 = 3.0
```

### 示例2: 中等难度调优项目
```
难度: 中等项目 (2.0)
状态: 调优中 (1.5)
负载 = 2.0 × 1.5 = 3.0
```

### 示例3: 高难度等待项目
```
难度: 高难项目 (3.0)
状态: 等待中（等客户） (0.3)
负载 = 3.0 × 0.3 = 0.9
```

### 示例4: 简单流失项目
```
难度: 简单项目 (1.0)
状态: 已流失 (0.1)
负载 = 1.0 × 0.1 = 0.1
```

## PE总负载计算示例

假设某PE有以下项目：
1. 高难项目 + 调优中 = 3.0 × 1.5 = 4.5
2. 中等项目 + 维护中 = 2.0 × 1.0 = 2.0
3. 高难项目 + 等待中（等客户） = 3.0 × 0.3 = 0.9
4. 简单项目 + 测试中 = 1.0 × 1.0 = 1.0
5. 中等项目 + 已流失 = 2.0 × 0.1 = 0.2

**总负载 = 4.5 + 2.0 + 0.9 + 1.0 + 0.2 = 8.6**

## 如何调整权重

### 方法1: 使用设置面板（推荐）

1. 点击页面右下角的 **⚙️ 设置按钮**
2. 调整各项权重和倍数
3. 点击 **"应用并重新计算"**
4. 系统会立即使用新权重重新计算所有负载

### 方法2: URL参数（高级）

在URL中添加参数：
```
?difficultWeight=4&mediumWeight=2&easyWeight=1&optimizeMultiplier=2
```

### 场景1: 强调高难度项目

```
等待中（等商务）: 0.1

```

## 排序功能

dashboard 支持按以下方式排序：
- **总项目数**: 按项目数量排序
- **工作负载**: 按加权负载排序（考虑难度和状态）
- **各客户状态**: 按特定状态的项目数排序

## 数据展示

### Header统计卡片
- **总工作负载**: 所有PE的加权负载总和
- **平均每人负载**: 总负载 ÷ PE数量

### PE详情卡片
- 显示每个PE的项目数和总负载
- 负载值保留一位小数

### 项目列表
- 每个项目显示难度等级
- 高难项目以红色 (#a22c29) 高亮显示

## 技术说明

- 权重通过URL查询参数传递给后端
- 后端实时计算负载，不存储权重
- 前端localStorage可用于保存用户偏好（未来功能）
- 权重变化后需重新加载数据才能生效

## 注意事项

⚠️ **重要**: 
- 调整权重后必须点击"应用并重新计算"
- 权重不会自动保存，刷新页面会恢复默认值
- 建议根据实际业务情况定期调整权重
- 所有权重和倍数必须 ≥ 0
